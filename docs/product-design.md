# AgentHub 产品设计文档

## 产品定位

AgentHub 是一个以 IM 聊天为核心交互范式的多 Agent 协作平台。用户像使用飞书或微信一样，通过新建对话、发送消息、@ Agent 和点击卡片操作，完成网页、代码、文档等产物的生成、预览、修改与发布。

## 目标用户

- 学生与开发者：用多个 AI 编程 Agent 快速完成比赛 Demo、课程项目和个人作品。
- 产品与设计同学：通过对话把需求拆成产品方案、原型、文案和视觉方向。
- 小团队：把需求讨论、AI 实现、评审反馈和部署发布沉淀在同一个上下文中。

## 核心场景

### 单聊模式

用户选择一个 Agent 作为聊天对象，例如 Codex。该模式适合目标明确的任务，如实现组件、修复报错、解释代码或生成测试。

### 群聊模式

用户在一个会话中 @ 多个 Agent，或把复杂目标直接交给 Orchestrator。Orchestrator 负责理解意图、拆解任务、分派 Agent，并在最后聚合结果。

### 产物内联

Agent 回复不只是文本，还可以带代码块、Diff 卡片、网页预览卡片、部署状态卡片和文档卡片。用户可以在聊天流中复制代码、应用 Diff、展开预览、引用消息继续修改。

### 上下文连续

每个会话保留完整历史。用户可以 pin 关键消息为长期上下文，后续 Agent 回复优先参考 pinned context，减少多轮修改中的需求漂移。

## 信息架构

- 左侧：工作台入口、会话列表、Agent 联系人。
- 中间：当前会话头部、pinned context、消息流、输入框和 @ 快捷栏。
- 右侧：产物工作台，包含预览、代码、版本三个 Tab。

## 关键流程

### 新建群聊任务

1. 用户点击“新建对话”。
2. 选择“群聊协作”。
3. 输入任务名称。
4. 平台创建会话并默认加入 Orchestrator、Codex、Designer、Deployer。
5. 用户发送任务，Orchestrator 开始拆解。

### 多 Agent 协作

1. 用户发送复杂需求。
2. Orchestrator 生成任务计划。
3. Designer 给出体验方案。
4. Claude Code 给出架构建议。
5. Codex 返回代码或 Diff。
6. OpenCode 补充本地运行和开源工具链方案。
7. Deployer 生成部署状态卡片。
8. Orchestrator 汇总结果和下一步建议。

### 一键应用 Diff

1. 用户在 Codex 消息中点击“应用 Diff”。
2. 平台更新右侧预览和代码视图。
3. 版本历史新增一条记录。
4. 消息流追加“Diff 已应用”的反馈。

## 产品亮点

- 像聊天一样指挥多个 Agent，降低工具切换成本。
- Agent 以联系人形式呈现，具备角色、头像、标签和自定义 Prompt。
- 产物内联在消息流里，用户能从“过程”直接到“结果”。
- Pinned context 让长对话更稳定，适合多轮迭代。
- 三栏结构便于答辩展示：左侧是对象，中间是过程，右侧是产物。

## 后续规划

- 引入 WebContainer 或沙箱执行，实现真实代码运行与测试。
- 支持多人协作、消息同步、冲突解决和权限控制。
- 支持桌面端本地文件访问与 Agent 进程管理。
- 支持移动端查看会话、审批部署和预览产物。
